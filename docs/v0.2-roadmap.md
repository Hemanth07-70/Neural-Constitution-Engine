# v0.2 Roadmap: Orchestration & Policy Extensibility

With the core deterministic governance kernel (v0.1.0-alpha) stabilized, the focus for v0.2 shifts toward real-world orchestration, dynamic policy language support, and telemetry.

## Theme 1: Dynamic Plugin Discovery & Packaging
- **Implement `importlib` Discovery**: Replace the stub in `EngineBuilder` with a robust dynamic plugin loader that scans configured directories.
- **Plugin Distribution**: Define a standard packaging format for third-party NCE plugins (e.g., wrapping them as standard PyPI packages with specific entry points).
- **Capability Granularity**: Expand `CapabilityRegistry` to support Transformer and Adapter capabilities for modifying requests in-flight.

## Theme 2: Advanced Policy Languages
- **Rego / OPA Integration**: Build an official `RuleEvaluator` plugin that compiles and evaluates rules written in Rego via the Open Policy Agent.
- **Semantic LLM Evaluators**: While the kernel remains LLM-free, v0.2 will introduce an official (but optional) plugin that securely calls external LLMs (e.g., Claude) to evaluate fuzzy, non-deterministic principles, mapping the results back into deterministic `RiskLevel` outputs.

## Theme 3: Execution Plan Schedulers (External)
- **LangGraph Adapter**: Create an official, out-of-tree adapter that converts an NCE `ExecutionPlan` into a compiled LangGraph `StateGraph` for actual execution.
- **Temporal.io Integration**: Provide bindings to map NCE plans to Temporal workflows for durable, distributed execution.

## Theme 4: Telemetry & Observability
- **Structured Logging**: Introduce an event-driven telemetry bus within the `EvaluationPipeline` to emit trace events at each stage.
- **OpenTelemetry (OTel)**: Provide an official plugin to seamlessly export audit records and pipeline traces to Datadog, Jaeger, or Prometheus.

## Theme 5: API & Webhooks
- **FastAPI Governance Server**: Release an optional, lightweight HTTP server wrapping the `Engine` to allow non-Python agents (e.g., TS/JS agents) to submit `DecisionRequests` over a RESTful API.
- **Webhook Postconditions**: Allow `PlanNode` postconditions to trigger generic webhooks upon successful execution.
